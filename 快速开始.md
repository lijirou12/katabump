## 💻 本地使用（Windows）

### 1. 配置账号

```bash
# 重命名模板文件
copy login.json.template login.json

# 编辑 login.json，填入你的账号信息
notepad login.json
```

格式：
```json
[
    {
        "username": "your_email@example.com",
        "password": "your_password"
    }
]
```

### 2. 修改 Chrome 路径（如果需要）

编辑 `renew.js` 第 12 行：

```javascript
const CHROME_PATH = "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe";
```

如果你的 Chrome 安装在其他位置，请修改此路径。

### 3. 运行脚本

```bash
# 直接运行
node renew.js

# 使用代理（如果需要）
# PowerShell:
$env:HTTP_PROXY="http://127.0.0.1:7890"
node renew.js

# CMD:
set HTTP_PROXY=http://127.0.0.1:7890
node renew.js
```

### 4. 查看结果

- **日志输出**：控制台会显示详细的执行过程
- **截图**：`photo/` 目录中保存了每个账号的截图

## ☁️ GitHub Actions 使用

### 1. Fork 本仓库

点击右上角的 `Fork` 按钮。

### 2. 配置 Secret

进入你的仓库：**Settings** → **Secrets and variables** → **Actions** → **New repository secret**

添加以下 Secret：

**必需：**
- `USERS_JSON`：你的账号信息（JSON 数组格式，压缩为一行）
  ```json
  [{"username":"user1@example.com","password":"pass1"},{"username":"user2@example.com","password":"pass2"}]
  ```

**可选：**
- `HTTP_PROXY`：代理地址（如果需要）
  ```
  http://127.0.0.1:7890
  或
  http://username:password@proxy.com:8080
  ```

- `TG_BOT_TOKEN` 和 `TG_CHAT_ID`：Telegram 通知（如果需要）

### 3. 启用 Workflow

进入 **Actions** 页面，启用 Workflow。

- 自动运行：每天北京时间 08:00
- 手动运行：点击 "Run workflow" 按钮

### 4. 查看结果

- **日志**：在 Actions 运行详情中查看
- **截图**：下载 `screenshots` artifact

## ❓ 常见问题

### Q1: 找不到 Turnstile checkbox？

**解决方法：**
- 确保网络正常，页面完全加载
- 增加等待时间（脚本已设置最多 30 秒）
- 查看截图确认 Turnstile 是否可见

### Q2: 点击了但验证失败？

**解决方法：**
- 已在新版本中增强，应该能解决
- 如果仍然失败，可能需要使用代理
- 查看 [TESTING_GUIDE.md](TESTING_GUIDE.md) 了解详细排查方法

### Q3: Chrome 启动失败？

**解决方法：**
```bash
# 检查 Chrome 路径是否正确
# 手动终止所有 Chrome 进程
taskkill /F /IM chrome.exe

# 重新运行
node renew.js
```

### Q4: 代理设置不生效？

**解决方法：**
```bash
# 检查代理格式
# 正确格式：http://host:port 或 http://user:pass@host:port

# 测试代理是否可用
curl -x http://127.0.0.1:7890 https://www.google.com
```
